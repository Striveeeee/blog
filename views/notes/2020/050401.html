        # 设置 HtmlWebpackPlugin

        但是，如果我们更改了我们的一个入口起点的名称，甚至添加了一个新的入口，会发生什么？会在构建时重新命名生成的 bundle，但是我们的 index.html 文件仍然引用旧的名称。让我们用 HtmlWebpackPlugin 来解决这个问题。

        安装插件，并调整webpack.config.js，以及在根目录新增index.html:

        yarn add html-webpack-plugin --dev
# or
npm install --save-dev html-webpack-plugin
        1
        2
        3

        由于HtmlWebpackPlugin会重写index.html模板，我们可以给它指定一个template，index.html:

        <!doctype html>
<html>
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width,initial-scale=1">
  <title><%= htmlWebpackPlugin.options.title %></title>
</head>
<body>
  <div id="app"></div>
</body>
</html>

        webpack.config.js:

        const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')

module.exports = {
  entry: {
    app: './src/index.js',
    show: './src/show.js'
  },
  plugins: [
    new HtmlWebpackPlugin({
      title: '管理输出由HtmlWebpackPlugin生成',
      template: 'index.html'
    })
  ],
  ...
}

        # 清理 /dist 文件夹

        我们发现dist文件夹现在由很多之前构建生成的无用文件，现在让我们通过lean-webpack-plugin来清理，使我们的项目更加整洁：

        npm install --save-dev clean-webpack-plugin
# or 
yarn add clean-webpack-plugin --dev
        1
        2
        3

        修改webpack.config.js:

        const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')

module.exports = {
  entry: {
    app: './src/index.js',
    show: './src/show.js'
  },
  plugins: [
    new CleanWebpackPlugin(),
    new HtmlWebpackPlugin({
      title: '管理输出由HtmlWebpackPlugin生成',
      template: 'index.html'
    })
  ],
  ...
}
