      # 选择一个开发工具

      每次代码发生变化都使用npm run build来构建是很麻烦的，webpack提供了以下三种方式来自动构建：

      1. webpack watch mode(webpack 观察模式)
      2. webpack-dev-server
      3. webpack-dev-middleware

      # 使用 watch mode(观察模式)

      在package.json的npm scripts中添加watch mode:

      "watch": "webpack --watch",
      1

      然后我们在命令行中运行:

      npm run watch
# or 
yarn watch
      1
      2
      3

      运行结束之后，并没有像build那样直接退出命令行。

      当去掉show.js的报错内容时，命令行内显示重新构建，但是dist下index.html也被清理了，在英文文档提供解决方法，修改webpack.config.js:

      const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')

module.exports = {
	...
  plugins: [
    new CleanWebpackPlugin({
      cleanStaleWebpackAssets: false
    }),
    ...
  ],
  ...
}

      # 调整文本编辑器

      不排除以后和使用其他编辑器作为开发工具，因此先记录下解决"safe write(安全写入)" 的问题方法:

      在一些常见的编辑器中禁用此功能，查看以下列表：

      Sublime Text 3：在用户首选项(user preferences)中添加 atomic_save: 'false'。

      JetBrains IDEs (e.g. WebStorm)：在 Preferences > Appearance & Behavior > System Settings 中取消选中 "Use safe write"。

      Vim：在设置(settings)中增加 :set backupcopy=yes。
